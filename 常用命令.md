# 常用命令

## 构建 minic

如果已有可用的 `build/minic`，这一节可以跳过。

```bash
cmake -B build-local -S . -G Ninja -DCMAKE_BUILD_TYPE=Debug -DCMAKE_CXX_COMPILER=clang++
cmake --build build-local --parallel
mkdir -p build
cp build-local/minic build/minic
```

构建完成后可确认编译器是否生成成功：

```bash
ls -l ./build/minic
./build/minic --help
```

## 本地跑测试集

本地测试脚本位置：

```bash
./tools/run-local-tests.sh
```

脚本当前的实际流程是：

1. `minic` 把测试用例编译成 ARM 汇编
2. `arm-linux-gnueabihf-gcc` 把 ARM 汇编和 `tests/std.c` 链接成可执行文件
3. `qemu-arm-static` 运行该 ARM 程序
4. 将程序输出和退出码与测试目录中的 `.out` 对比

### 1. 跑全部测试

```bash
bash ./tools/run-local-tests.sh
```

这条命令会依次运行：

- `tests/2023_function`
- `tests/2025_function`
- `tests/2025_performance`

### 2. 跑单个测试集

运行 2023 功能测试：

```bash
bash ./tools/run-local-tests.sh 2023
```

运行 2025 功能测试：

```bash
bash ./tools/run-local-tests.sh 2025
```

运行 2025 性能测试：

```bash
bash ./tools/run-local-tests.sh 2025_perf
```

也支持使用完整目录别名：

```bash
bash ./tools/run-local-tests.sh 2023_function
bash ./tools/run-local-tests.sh 2025_function
bash ./tools/run-local-tests.sh 2025_performance
```

### 3. 跑单个测试用例

指定“测试集 + 用例名”：

```bash
bash ./tools/run-local-tests.sh 2023 2023_func_00_main
bash ./tools/run-local-tests.sh 2023 2023_func_01_var_defn2
bash ./tools/run-local-tests.sh 2025 2025_func_000_main
bash ./tools/run-local-tests.sh 2025_perf 2025_perf_01_mm1
```

也可以直接只写用例名，脚本会自动推断所属测试集：

```bash
bash ./tools/run-local-tests.sh 2023_func_00_main
bash ./tools/run-local-tests.sh 2025_func_000_main
bash ./tools/run-local-tests.sh 2025_perf_01_mm1
```

### 4. 与 CI 当前一致的测试命令

当前 CI 只跑 `2023_function` 中前两个测试，对应本地命令为：

```bash
bash ./tools/run-local-tests.sh 2023 2023_func_00_main
bash ./tools/run-local-tests.sh 2023 2023_func_01_var_defn2
```

### 5. 常见排查命令

检查脚本语法：

```bash
bash -n ./tools/run-local-tests.sh
```

检查当前 Docker 或宿主机架构：

```bash
uname -m
arch
```

确认 ARM 交叉编译与运行工具是否存在：

```bash
command -v arm-linux-gnueabihf-gcc
command -v qemu-arm-static
```

### 6. 手动跑某个测试的完整链路

以 `2023_func_00_main` 为例：

```bash
./build/minic -S -O1 -o /tmp/2023_func_00_main.s ./tests/2023_function/2023_func_00_main.c
arm-linux-gnueabihf-gcc -g -static -o /tmp/2023_func_00_main /tmp/2023_func_00_main.s ./tests/std.c
qemu-arm-static /tmp/2023_func_00_main
printf "\nEXIT=%s\n" $?
```

如果测试带输入文件，可手动重定向：

```bash
qemu-arm-static /tmp/your_test < ./tests/2025_function/your_test.in
printf "\nEXIT=%s\n" $?
```
