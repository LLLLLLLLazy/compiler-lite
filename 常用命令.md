# 常用命令

## 构建 minic

如果已有可用的 `build/minic`，这一节可以跳过。

```bash
cmake -B build-local -S . -G Ninja -DCMAKE_BUILD_TYPE=Debug -DCMAKE_CXX_COMPILER=clang++
cmake --build build-local --parallel
mkdir -p build
cp build-local/minic build/minic
```

构建完成后可确认编译器是否生成成功：

```bash
ls -l ./build/minic
./build/minic --help
```

## 本地跑测试集

本地测试脚本位置：

```bash
./tools/run-local-tests.sh
```

脚本支持两类维度：

1. 前端选择
2. 测试验证模式

### 1. 前端选择

默认使用 `ANTLR`：

```bash
bash ./tools/run-local-tests.sh 2023 2023_func_00_main
```

### 2. 测试验证模式

默认模式是 `asm`，即：

1. `minic` 生成 ARM 汇编
2. `arm-linux-gnueabihf-gcc` 交叉编译
3. `qemu-arm-static` 运行
4. 将程序输出和退出码与 `.out` 对比

```bash
MINIC_TEST_MODE=asm bash ./tools/run-local-tests.sh 2023 2023_func_00_main
```

#### 2.1 DragonIR 功能测试

对输出的线性 IR（DragonIR）通过 `IRCompiler` 进行功能测试验证：

```bash
MINIC_TEST_MODE=ir bash ./tools/run-local-tests.sh 2023 2023_func_00_main
```

这条命令会执行：

1. `./build/minic -S -I -o xxx.ir xxx.c`
2. `tools/IRCompiler/.../IRCompiler -R xxx.ir`
3. 将解释执行的输出和退出码与 `.out` 对比

#### 2.2 汇编功能测试

对输出的汇编通过交叉编译与 qemu 运行进行功能测试验证：

```bash
MINIC_TEST_MODE=asm bash ./tools/run-local-tests.sh 2023 2023_func_00_main
```

#### 2.3 线性 IR 解释执行测试

这和 `ir` 模式一致，本仓库当前就是通过 `IRCompiler` 对生成的 DragonIR 做解释执行：

```bash
MINIC_TEST_MODE=ir bash ./tools/run-local-tests.sh 2023 2023_func_00_main
```

#### 2.4 AST 语法树测试

对输出的 AST 图像进行测试验证：

```bash
MINIC_TEST_MODE=ast bash ./tools/run-local-tests.sh 2023 2023_func_00_main
```

当前 AST 模式的验证规则是：

1. `minic` 成功生成 AST 图片
2. 输出文件非空
3. 如果测试目录下存在同名参考图片 `xxx.png`，则做精确比对

#### 2.5 一次跑完多种验证

一次对同一个测试用例同时跑汇编、IR、AST 三种验证：

```bash
MINIC_TEST_MODE=all bash ./tools/run-local-tests.sh 2023 2023_func_00_main
```

### 3. 跑全部测试

默认前端为 `ANTLR`，默认验证模式为 `asm`：

```bash
bash ./tools/run-local-tests.sh
```

这条命令会依次运行：

- `tests/2023_function`
- `tests/2025_function`
- `tests/2025_performance`

### 4. 跑单个测试集

运行 2023 功能测试：

```bash
bash ./tools/run-local-tests.sh 2023
```

运行 2025 功能测试：

```bash
bash ./tools/run-local-tests.sh 2025
```

运行 2025 性能测试：

```bash
bash ./tools/run-local-tests.sh 2025_perf
```

也支持使用完整目录别名：

```bash
bash ./tools/run-local-tests.sh 2023_function
bash ./tools/run-local-tests.sh 2025_function
bash ./tools/run-local-tests.sh 2025_performance
```

### 5. 跑单个测试用例

指定“测试集 + 用例名”：

```bash
bash ./tools/run-local-tests.sh 2023 2023_func_00_main
bash ./tools/run-local-tests.sh 2023 2023_func_01_var_defn2
bash ./tools/run-local-tests.sh 2025 2025_func_000_main
bash ./tools/run-local-tests.sh 2025_perf 2025_perf_01_mm1
```

也可以直接只写用例名，脚本会自动推断所属测试集：

```bash
bash ./tools/run-local-tests.sh 2023_func_00_main
bash ./tools/run-local-tests.sh 2025_func_000_main
bash ./tools/run-local-tests.sh 2025_perf_01_mm1
```

### 6. 与 CI 当前一致的测试命令

当前 CI 只跑 `2023_function` 中前两个测试，默认前端为 ANTLR，验证模式为 `asm`：

```bash
MINIC_TEST_MODE=asm bash ./tools/run-local-tests.sh 2023 2023_func_00_main
MINIC_TEST_MODE=asm bash ./tools/run-local-tests.sh 2023 2023_func_01_var_defn2
```

### 7. 常见排查命令

检查脚本语法：

```bash
bash -n ./tools/run-local-tests.sh
```

检查当前 Docker 或宿主机架构：

```bash
uname -m
arch
```

确认 ARM 交叉编译与运行工具是否存在：

```bash
command -v arm-linux-gnueabihf-gcc
command -v qemu-arm-static
```

确认 IR 运行器是否存在：

```bash
find ./tools/IRCompiler -type f | sort
```

### 8. 手动跑某个测试的完整链路

#### 8.1 ANTLR + DragonIR + IRCompiler

```bash
./build/minic -S -I -o /tmp/2023_func_00_main.ir ./tests/2023_function/2023_func_00_main.c
./tools/IRCompiler/Linux-x86_64/Ubuntu-22.04/IRCompiler -R /tmp/2023_func_00_main.ir
printf "\nEXIT=%s\n" $?
```

#### 8.2 ANTLR + ARM 汇编 + qemu

```bash
./build/minic -S -O1 -o /tmp/2023_func_00_main.s ./tests/2023_function/2023_func_00_main.c
arm-linux-gnueabihf-gcc -g -static -o /tmp/2023_func_00_main /tmp/2023_func_00_main.s ./tests/std.c
qemu-arm-static /tmp/2023_func_00_main
printf "\nEXIT=%s\n" $?
```

#### 8.3 ANTLR + AST 图片输出

```bash
./build/minic -S -T -o /tmp/2023_func_00_main.png ./tests/2023_function/2023_func_00_main.c
file /tmp/2023_func_00_main.png
ls -l /tmp/2023_func_00_main.png
```
